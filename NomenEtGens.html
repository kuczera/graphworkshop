01

Das Projekt Nomen et Gens

  • Das Projekt „Nomen et Gens“ (NeG) zielt darauf ab, alle schriftlich belegten Namen und Personen Kontinentaleuropas in den vier Jahrhunderten vor Karl dem Großen (also von 400 bis 800 nach Christus) zu erfassen.
  • Die Datenbank des Projekts geht auf ein erfolgreich abgeschlossenes DFG-Projekt zurück und wird aktuell von den Projektbeteiligten weiter betreut und sukzessive ausgebaut.
  • Neben den Quellen der Personennennung, den unterschiedlichen Namensformen usw. werden auch die Verwandtschaftsbeziehungen zwischen identifizierten Personen in der Datenbank abgelegt.
  • Bei einem Personendatensatz werden dann jeweils die Verwandtschaftsbeziehungen aufgelistet, so dass man sich ggf. jeweils von Person zu Person durchklicken muss, bis man am Ziel angelangt ist (Beispiel Karl der Große mit der ID 7404).

02

Modellierung von Verwandtschaft

  • In der Datenbank werden bis zu 16 verschiedene Verwandtschaftsbeziehungen gespeichert, wie z. B. Bruder, Schwester, Sohn, Tochter, Vater, Mutter, Onkel oder Nichte (Beispiel).
  • Welche Typen von Kanten werden für die Modellierung von Verwandtschaftsbeziehungen mindestens benötigt ?
  • Die Testdatenbank findet man unter diesem Link. Sie stellt einen Proof of Concept dar und erhebt keinen Anspruch auf Vollständigkeit.

03

Die Urenkel Pippins II.

mit ihren Pfaden

  • Mit folgendem Cypher-Query findet man Pippin II.: 
    // Pippin II.
MATCH (P:Person {nid:'7212'})
RETURN *;
  • Mit folgendem Cypher-Query findet man die Kinder Pippins II.: 
    // Die Kinder Pippins II.
MATCH (P:Person {nid:'7212'})-[:KIND]->(K)
RETURN *;
  • Mit folgendem Cypher-Query finden sich schließlich die Urenkel Pippins II.: 
    // Die Urenkel Pippins II.
MATCH (P:Person {nid:'7212'})-[:KIND]->(K)-[:KIND]->(E)-[:KIND]->(UE)
RETURN *;

04

Die Neffen von Karl Martell

  • Die Neffen von Karl Martell sind die Enkel Pippins II. ohne die Kinder von Karl Martell. Mit folgendem Cypher-Query findet man Karl Martell: 
    // Karl Martell
MATCH (P:Person {nid:'23022'})
RETURN *;
  • Mit folgendem Cypher-Query findet man die Neffen von Karl Martell: 
    // Die Neffen von Karl Martell
MATCH (P:Person {nid:'23022'})<-[:KIND]-(E)
-[:KIND]->(B)-[:KIND]->(N {Geschlecht:'m'})
RETURN *;
  • Mit folgendem Cypher-Query findet man NUR die Stiefneffen von Karl Martell: 
    // Die Neffen von Karl Martell
MATCH (P:Person {nid:'23022'})<-[:KIND]-(E)-[:KIND]->(B)-[:KIND]->(N)
WHERE B.nid <> P.nid
RETURN *;
  • Mit dem Zusatz "WHERE B.nid <> P.nid" wird der Pfad zurück über Karl Martell zu seinen Kindern ausgeschlossen.

05

Sind Berchar und Karl der Große verwandt ?

  • In der Datenbank gibt es die Person Berchar. Berchar war ein Hausmeier König Theuderichs III.
  • Mit folgendem Cypher-Query findet man Berchar: 
    // Berchar
MATCH (B:Person {nid:'7119'})
RETURN *;
  • In der Graphdatenbank neo4j wird für eine solche Fragestellung eine Shortest_Path-Abfrage verwendet, die den kürzesten möglichen Weg zwischen zwei Knoten zurück liefert, sofern es einen gibt.
  • Der folgende cypher-Befehl liefert den Pfad zwischen dem Personenknoten Karls des Großen mit der NeG-ID 7404 und dem Personenknoten von Berchar mit der NeG-ID 7119. Dabei wird die Länge des abzufragenden Pfades auf 15 Kanten begrenzt. 
    // shortest_path
MATCH
(KdG:Person { nid:'7404' }),
(Berchar:Person { nid:'7119' }),
p = shortestPath((KdG)-[*..15]-(Berchar))
RETURN p;
