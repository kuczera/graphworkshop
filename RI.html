01

Das Projekt Regesta Imperii

  • Die Regesta Imperii sind ein Projekt aus dem Bereich der mediävistischen Grundlagenforschung.
  • Im Rahmen des Projekts werden deutschsprachige Inhaltsangaben von Kaiser-, König- und Papsturkunden erstellt, die die handelnden Personen, die Orte und die rechtlich relevanten Inhalte zusammenfassen.
  • Im Jahre 1829 von seinem Initiator Johann Friedrich Böhmer ursprünglich als Vorarbeit für die Monumenta Germaniae Historica angelegt, entwickelte sich das Projekt mit einem erweiterten Regestenkonzept zu einem eigenständigen historischen Grundlagenwerk fort.
  • Im Rahmen eines DFG-Projekts wurden die Regesta Imperii von 2001 bis 2006 retrodigitalisiert und stehen seither als Volltext im Internet zur Verfügung.
  • Momentan werden die teilweise schon digital vorliegenden Register aufgearbeitet und für die Publikation im Internet vorbereitet.
  • In dieser Präsentation werden Erschließungsperspektiven anhand der Register der Regesten Kaiser Heinrichs IV. vorgestellt.

02

Das Regest im Graphen

  • Für die Erstellung der Graphdatenbank werden die Regesten aus unserem Datenrepositorium als CSV-Daten exportiert und anschließend in die Graphdatenbank importiert. Dieser Import ist nicht fehlerfrei und stellt einen Proof-of-Concept dar (Beispiel 1, Beispiel 2).
  • Aus den Regesteninformationen werden dabei Knoten und Kanten erstellt. Der Regestentext und die Überlieferung werden als Eigenschaften (oder properties) der Knoten abgespeichert und bleiben damit für die Volltextrecherche über cypher verfügbar.
  • Beim Importprozess wird das erste Verb des Regests identifiziert und lemmatisiert und anschließend mit den jeweiligen Regesten verbunden. Damit wird die Beobachtung des Herrscherhandelns in Raum und Zeit möglich.
  • Im Überlieferungsteil sind die Literaturhinweise meist mit der Literaturdatenbank der Regesta Imperii, dem RI-Opac verlinkt. Da diese Links eindeutig sind, können sie mit cypher-queries identifiziert und als eigener Knotentyp in der Graphdatenbank angelegt und für die Erstellung von Zitationsnetzwerken verwendet werden.
  • Das Register wird in einem eigenen Prozess importiert und mit den Regesten verknüpft. Beim Import bleibt die Hierarchie des Registers erhalten.

03a

Personenbeziehungen durch Registerauswertung

  • Die Registeraufbereitung ist Teil eines kleinen Forschungsprojekts mit Prof. Lubich (Bochum) in dessen Rahmen Auswertungsperspektiven für Register in Graphdatenbanken erörtert werden.
  • Ausgangspunkt ist Graf Robert von Flandern, den man mit folgendem cypher-query findet: 
    // Robert II. von Flandern
MATCH (n:IndexPerson) WHERE n.registerId = 'H4P01822'
RETURN *;
  • Welche weiteren Personen sind in den Regesten gemeinsam mit Robert genannt ? 
    // Robert II. von Flandern mit Netzwerk
MATCH (n:IndexPerson)-[:PERSON_IN]->(r:Regesta)<-[:PERSON_IN]-
(m:IndexPerson)
WHERE n.registerId = 'H4P01822'
RETURN *;

03b

Personenbeziehungen durch Registerauswertung

  • Eine weitere Ebene dazu:
    // Robert II. von Flandern mit Netzwerk und Herrscherhandeln (viel)
MATCH
(n1:IndexPerson)-[:PERSON_IN]->(r1:Regesta)<-[:PERSON_IN]-
(n2:IndexPerson)-[:PERSON_IN]->(r2:Regesta)<-[:PERSON_IN]-
(n3:IndexPerson)
WHERE n1.registerId = 'H4P01822'
RETURN *;
  • Welche Beziehungen bestehen zwischen Robert II. von Flandern und Herzog Heinrich von Niederlothringen in der Graphdatenbank ? 
    // Robert II. von Flandern und Herzog Heinrich von Niederlothringen mit Netzwerk
MATCH
(n:IndexPerson)-[:PERSON_IN]->
(r:Regesta)<-[:PERSON_IN]-(m:IndexPerson)
WHERE n.registerId = 'H4P01822'
AND m.registerId = 'H4P00926'
RETURN *;

03c

Personenbeziehungen durch Registerauswertung

  • Gibt es Personen, mit denen beide in Regesten gemeinsam genannt sind ... 
    MATCH
(startPerson:IndexPerson)-[:PERSON_IN]->
(regest:Regesta)<-[:PERSON_IN]-(endPerson:IndexPerson)
WHERE startPerson.registerId in ['H4P01822', 'H4P00926']
RETURN *;
  • Verlagerung der Regesten in APPEARS_WITH-Kanten ... 
    MATCH
(startPerson:IndexPerson)-[:PERSON_IN]->
(regest:Regesta)<-[:PERSON_IN]-(endPerson:IndexPerson)
WHERE startPerson.registerId in ['H4P01822', 'H4P00926']
WITH startPerson, endPerson, count(regest) as anzahl,
collect(regest.ident) as idents
CALL apoc.create.vRelationship(startPerson, "APPEARS_WITH",
{anzahl:anzahl, regesten:idents}, endPerson) YIELD rel
RETURN startPerson, endPerson, rel;

03d

Personenbeziehungen durch Registerauswertung

  • Und nun nur die interessanten Regesten als Ergbenis: 
    MATCH
(startPerson:IndexPerson)-[:PERSON_IN]->
(regest1:Regesta)<-[:PERSON_IN]-(middlePerson:IndexPerson)
-[:PERSON_IN]->(regest2:Regesta)<-[:PERSON_IN]-(endPerson:IndexPerson)
WHERE startPerson.registerId in ['H4P00926']
AND endPerson.registerId in ['H4P01822']
RETURN DISTINCT startPerson.name1, regest1.ident, regest1.text,
middlePerson.name1, regest2.ident, regest2.text, endPerson.name1;

04

Statistische Betrachtungen

  • Herrscherhandeln ausgezählt 
    MATCH (n:Lemma)<-[h:ACTION]-(m:Regesta)
RETURN n.lemma, count(h) as ANZAHL ORDER BY ANZAHL desc LIMIT 25;
  • Herrscherhandeln pro Ausstellungsort ausgezählt 
    // Herrscherhandeln pro Ausstellungsort
MATCH (n:Lemma)<-[h:ACTION]-(:Regesta)-[:PLACE_OF_ISSUE]->(p:Place)
WHERE p.normalizedGerman IS NOT NULL
RETURN p.normalizedGerman, n.lemma, count(h) as ANZAHL ORDER BY ANZAHL desc LIMIT 25;
  • Welcher Literaturtitel wird am meisten zitiert ? 
    MATCH (n:Reference)<-[r:REFERENCES]-(m:Regesta)
RETURN n.title, count(r) as ANZAHL ORDER BY ANZAHL desc LIMIT 25;

05

Herrscherhandeln und Anwesenheit

  • Welche Personen sind bei welchen Herrscherhandlungen wie häufig 
    MATCH (p:IndexPerson)-[:PERSON_IN]-(r:Regesta)-[:ACTION]-(l:Lemma)
RETURN p.name1, l.lemma, count(l) AS Anzahl ORDER BY p.name1, Anzahl DESC;
  • Regesten 200 km rund um Augsburg 
    // Entfernungen von Orten berechnen lassen
MATCH (n:Place)
WHERE n.normalizedGerman = 'Augsburg'
WITH n.latLong as point
MATCH (r:Regesta)
WHERE distance(r.latLong, point) < 200000
AND r.placeOfIssue IS NOT NULL
AND r.placeOfIssue <> 'Augsburg'
RETURN r.ident, r.placeOfIssue,
distance(r.latLong, point) AS Entfernung
ORDER BY Entfernung;
